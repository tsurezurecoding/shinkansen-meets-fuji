# 新幹線 meets 富士山 / Shinkansen Meets Fuji 要件定義

## サービス名

- 日本語名: 新幹線 meets 富士山
- 英語名: Shinkansen Meets Fuji
- リポジトリ名候補: `shinkansen-meets-fuji`

## サービス概要

東海道・山陽新幹線に乗る人向けに、車窓から富士山が見える見どころ時間帯を案内するWebアプリ。

ユーザーは上り / 下りを選び、列車カードから号数を選択する。アプリはその列車の時刻表データをもとに、富士山の見どころ時間帯、停車駅タイムライン、富士山が見える方向を表示する。

## 想定ユーザー

- 日本国内の旅行者
- インバウンド観光客
- 初めて東海道新幹線に乗る人
- 富士山を写真に撮りたい人
- 車窓から富士山を見逃したくない人

## MVP範囲

- 静的Webアプリ
- GitHub Pagesで公開可能
- 日本語 / 英語対応
- ログインなし
- データベースなし
- 外部APIなし
- 保存済み時刻表データを利用
- PWA対応
- ブラウザを閉じた状態のPush通知は対象外

## 主な画面構成

### ヒーロー

- タイトル: 新幹線 meets 富士山
- 英語: Shinkansen Meets Fuji
- 説明文: 列車を選ぶと、富士山の見どころ時間帯を確認できます。
- 言語切替: 日本語 / English
- 背景画像: `shinkansenmeetsfuji.png`

### 列車選択

- 上り / 下り切替
- 新横浜基準の時間ジャンプボタン
- Cover Flow風の列車カード
- 左右キー、フリック、タップ/クリックで選択

### タイムライン

- 選択中の列車名
- 時刻 / 駅 / メモ
- のぞみ停車駅ベースのタイムライン
- 富士山見どころ行を時系列に差し込み
- 見どころ行に「進行方向の右側 / 左側に見える目安」を表示
- 通知情報と通知テストを統合

### 注意事項

- 天候、遅延、窓枠、防音壁などで見え方が変わる旨を表示

## 入力仕様

ユーザー入力:

- 上り / 下り
- 列車カード選択

削除した入力:

- 駅名と出発時刻
- 車両タイプ
- 座席番号
- 座席側選択

## 列車カード仕様

表示対象:

- 保存済み時刻表データの全列車

並び順:

- 新横浜の発車/到着時刻基準でソート
- 新横浜の時刻がない場合は、富士山見どころ時刻と新横浜オフセットから推定
- 最後の保険として始発時刻

カード表示:

- 列車種別
- 号数
- 始発駅と時刻
- 行き先
- 富士山見どころ時間帯

Cover Flow操作:

- 中央カードが選択中
- 左右に前後カードを薄く表示
- 左右キーで移動
- フリック/スワイプで移動
- カードを押す/タップするとその列車へ遷移
- ホバーだけでは切り替えない

## 時間ジャンプ仕様

- 列車カード上に時間ジャンプボタンを表示
- 新横浜基準
- 4時間刻み
- 例: `04:00`, `08:00`, `12:00`, `16:00`, `20:00`
- 押すと、その時間帯の最初の列車へジャンプ

## 見どころ計算

中心:

- 新富士駅付近
- 三島〜新富士区間

表示:

- 単一時刻ではなく時間帯
- 例: `08:12-08:17`

方向:

- 下り: 進行方向の右側
- 上り: 進行方向の左側

補正:

- 新富士停車列車は新富士時刻を優先
- 新富士時刻がない場合は主要駅オフセットから推定
- 新横浜、品川、東京、名古屋、京都、新大阪などの時刻から補正

## 時刻表データ

保存先:

- `data/timetable.json`
- `data/timetable.js`

用途:

- `timetable.json`: 再利用・更新用の元データ
- `timetable.js`: `file://` 直開きでも動くためのブラウザ読込用データ

現在の保存データ:

- 546列車
- 32駅
- 下り: 275
- 上り: 271

取得元:

- JR East 東京駅 下り
- JR East 品川駅 下り
- JR East 品川駅 上り
- JR East 新横浜駅 下り
- JR East 新横浜駅 上り
- JR東海公式時刻表ページ
- JR東海ツアーズ座席ガイド

注意:

- 基本時刻表ベース
- 臨時列車、運転日注意、季節変更、遅延は完全反映ではない
- 今後もJSON更新で拡張可能

## PWA仕様

追加ファイル:

- `manifest.json`
- `sw.js`

対応内容:

- ホーム画面追加
- standalone表示
- アプリ本体のキャッシュ
- CSS/JS/画像/時刻表データのキャッシュ
- 弱電波・オフライン時の表示補助

対象外:

- ブラウザを閉じた状態のPush通知
- サーバー側通知予約
- ユーザーごとの通知保存

将来Push通知をやる場合:

- Cloudflare Workers / Firebase / Supabaseなどが必要
- Push購読情報保存
- VAPIDキー
- 通知送信ジョブ
